        <div id='description'>
          <p>
          Specify this <tt>acts_as</tt> extension if you want to model a tree
          structure by providing a parent association and a children association.
          This requires that you have a foreign key column, which by default is
          called <tt>parent_id</tt>.
          </p>
          <pre>class Category &lt; ActiveRecord::Base&#x000A;  acts_as_sane_tree :order =&gt; &quot;name&quot;&#x000A;end&#x000A;&#x000A;Example:&#x000A;root&#x000A; \_ child1&#x000A;      \_ subchild1&#x000A;      \_ subchild2&#x000A;&#x000A;root      = Category.create(&quot;name&quot; =&gt; &quot;root&quot;)&#x000A;child1    = root.children.create(&quot;name&quot; =&gt; &quot;child1&quot;)&#x000A;subchild1 = child1.children.create(&quot;name&quot; =&gt; &quot;subchild1&quot;)&#x000A;&#x000A;root.parent   # =&gt; nil&#x000A;child1.parent # =&gt; root&#x000A;root.children # =&gt; [child1]&#x000A;root.children.first.children.first # =&gt; subchild1</pre>
          <p>
          The following class methods are also added:
          </p>
          <ul>
          <li><tt>nodes_within?(src, chk)</tt> - Returns true if chk contains any nodes
          found within src and all ancestors of nodes within src

          </li>
          <li><tt>nodes_within(src, chk)</tt> - Returns any matching nodes from chk found
          within src and all ancestors within src

          </li>
          <li><tt>nodes_and_descendents(*args)</tt> - Returns all nodes and descendents
          for given IDs or records. Accepts multiple IDs and records. Valid options:

          <ul>
          <li>:raw - No Hash nesting

          </li>
          <li>:no_self - Will not return given nodes in result set

          </li>
          <li>{:depth => n} - Will set maximum depth to query

          </li>
          </ul>
          </li>
          </ul>

                <p>
                Configuration options are:

                <ul>
                <li><tt>foreign_key</tt> - specifies the column name to use for tracking of the

                </li>
                <li><tt>order</tt> - makes it possible to sort the children according to this
                SQL snippet.

                </li>
                <li><tt>counter_cache</tt> - keeps a count in a <tt>children_count</tt> column
                if set to <tt>true</tt> (default: <tt>false</tt>).

                </li>
                </ul>
