          <h2>acts_as_sane_tree</h2>
          <h4>(Building trees with a dash of sanity)</h4>
          <p>
          This is a drop in replacement for acts_as_tree on systems with Postgresql
          >= 8.4
          </p>
          <h2>What this provides</h2>
          <p>
          A fast way to build trees.
          </p>
          <h2>Requirements</h2>
          <ul>
          <li>PostgreSQL version >= 8.4

          </li>
          <li>ActiveRecord

          </li>
          </ul>
          <h2>Configuration</h2>
          <p>
          Same as acts_as_tree. Basically: Specify a parent_id or the column that
          holds the parent ID information.
          </p>
          <h2>Extras</h2>
          <p>
          A few extras are provided. Of note are:
          </p>
          <ul>
          <li>depth -> depth from root of the current node

          </li>
          <li>descendents -> all descendents of the current node (provided in either flat
          array or nested hash)

          </li>
          <li>nodes_within?(src, chk) - Returns true if chk contains any nodes found
          within src and all ancestors of nodes within src

          </li>
          <li>nodes_within(src, chk) - Returns any matching nodes from chk found within
          src and all ancestors within src

          </li>
          <li>nodes_and_descendents(*args) - Returns all nodes and descendents for given
          IDs or records.

          </li>
          </ul>
